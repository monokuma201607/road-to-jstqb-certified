# Weak Point Targeted Mode (弱点特化型・集中演習)

このコマンド（`/07_weak_point_mode`）は、ログやカバレッジ分析（例: `Continuous_Summary.md` や直近のテスト結果）で「弱点（🟨）」とされている項目（特定のテスト技法、状態遷移、ドメイン分析など）をピンポイントで潰し、確実に「習得済み（🟩）」へと引き上げるための集中トレーニングモードです。

## 実施フロー

1.  **Weakness Scan**: 
    - ユーザーの直近の学習ログや `Continuous_Summary.md` に記載された課題を読み込み、現在の弱点を特定します。
    - ユーザーに「本日の重点対策トピック（例: 直交表とペアワイズ法の使い分け）」を提示し、合意を得ます。

2.  **Principle Deep Dive (原理の深掘り)**:
    - 選択されたトピックについて、シニアQAマネージャーの視点から「なぜこの技法が必要なのか」「どのような制約やリスクがある場合に他の技法より優れているのか（K4視点）」を深く解説します。
    - 単なる手順の暗記ではなく、「AI駆動のテスト生成プロンプトに落とし込むならどう定義するか」というアーキテクト視点を交えて本質的な理解を促します。

3.  **Precision Drilling (精密シナリオ演習)**:
    - JSTQB AL TAで最も迷いやすい「技法の選択・組み合わせ（例: 状態遷移テストのNスイッチカバレッジと、デシジョンテーブルのハイブリッドなど）」に焦点を当てた、高難度（K4レベル）の長文シナリオ問題を出題します。
    - 単純な計算ミスを誘う問題ではなく、「ビジネス要件からリスクを読み取る力」を問う問題を連続で実施します。

4.  **Verification & Promotion**:
    - ユーザーの解答と解説ディスカッションを通し、分析力（K4）が十分なレベルに達したと判断した場合、弱点ステータスを「習得済み（🟩）」に更新することを提案します。
    - 本日の成長記録を `04_output_generator` に従って `Daily_Lessons/` に保存します。

## 出題・解説のガイドライン
- 問題は「知識の確認」にとどまらず、「実務プロジェクトでの最適な判断」をシミュレートするものにします。
- ユーザーが誤った技法を選択した場合、単に正解を教えるのではなく、「もしその技法でリリースした場合、本番環境でどのような欠陥がすり抜けるリスクがあるか」というQAとしての影響度（Impact）を考えさせる解説を行います。
