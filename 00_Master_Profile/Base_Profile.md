# Base Profile: JSTQB Foundation Level Specialist

## 1. 最終ゴール (Final Goal)
- **目標試験**: JSTQB Certified Tester Foundation Level (Version 4.0) 合格
- **目標レベル**: プロフェッショナル・テスト技術者
    - シラバスの用語（エラー・欠陥・故障など）を厳格に使い分けられる。
    - 7つのテスト原則を実務のコンテキストに当てはめて説明できる。
    - テスト技法（ブラックボックス/ホワイトボックス）を用いて、適切なテストケースを設計できる（K3レベル）。
    - 開発ライフサイクル（アジャイル/シーケンシャル）に応じたテストアプローチを提案できる。

## 6. 90%突破のための戦略分析 (Strategy for 90%+)
現在の正答率 70-80% から 90%（安定して36問/40問正解レベル）へ引き上げるための課題と対策：

### 課題1: 用語定義の「厳密な」境界線の言語化
- **分析**: 「確認 vs リグレッション」「モニタリング vs コントロール」「UAT vs OAT」など、似た概念での失点が継続している。
- **対策**: シラバスの用語の「何が決定的な違いか」を10文字程度で言語化する訓練を行う（例：「コントロール＝修正アクション」）。

### 課題2: 問題文の「制約条件」の完全網羅
- **分析**: 「有効な境界値のみ」「不適切なもの」「2点 vs 3点」といった条件の読み落としによる失点がある。
- **対策**: 解答前に語尾と条件（有効/無効、肯定/否定）を指差し確認する。

### 課題3: K3レベル（設計技法）の最小化ロジック
- **分析**: 状態遷移や判定網羅において、複数のパスを1つのケースで網羅する「最小化」でミスが発生しやすい。
- **対策**: 第4章の技法は、単に値を出すだけでなく「最小のケース数で網羅する組み合わせ」をパターンとして習得する。

### 課題4: JSTQB Version 4.0 特有の新概念
- **分析**: テストプロセスの並行性や、メンテナンステストのトリガー（廃棄時を含む）など、旧版や一般常識と異なる点での失点。
- **対策**: シラバス V4.0 の「1.4.2 テストプロセス」を重点的に再読する。
