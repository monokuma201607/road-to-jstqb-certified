# Base Profile: AI-Driven QA Engineer & AL TA Candidate

## 1. 最終ゴール (Final Goal)
- **短中期目標**: JSTQB Advanced Level Test Analyst (AL TA) 合格
- **中長期目標**: JSTQB Advanced Level Test Manager (AL TM) 取得・「AIを活用したテストプロセス改善」のリーダー
- **目標レベル**: 
    - AL TAの要件である「複雑なコンテキストにおける最適なテスト技法の分析・選択とコスト評価（K4）」ができる。
    - システムのアーキテクチャやドメインの制約を紐解き、直交表、ペアワイズ、ユースケース等を用いてテストケースを極限まで最適化できる。
    - 上記の「高度なテスト設計ロジック」をLLM（AI）のプロンプトに昇華させ、テスト設計〜実行の自動化プロセスを構築できる。

## 2. 学習スタイル (Learning Style)
- **メイン教材**: 
    - JSTQB Advanced Level Test Analyst シラバス
    - 海外/公式のサンプル長文シナリオ問題
- **トレーニング手法**:
    - **Advanced Scenario Drill**: 複数の条件や利害対立が絡む長文シナリオを読み解き、「最も妥当なアプローチはどれか」とその根拠をディスカッション（壁打ち）する。
    - **Optimization Exercise**: ブラックボックス技法の高度化（ドメイン分析、Nスイッチカバレッジ等）を用いた「最小テストケース構成」の数理的演習。
    - **AI Architecture Talk**: 「このプロセスをAIで代替させるならどう設計するか？」というマネージャー視点のレビューを常に行う。
- **管理手法**: Git による学習ログ（`01_Progress_Log/`）の継続と、『AIプロンプト/テスト手法テンプレート集』の資産化。

## 5. 重点課題 (Focus Points for AL TA & TM)
- [ ] **シナリオ読解力（K4）の向上**: 現場の制約（スケジュール、リスク、コスト）から、「全網羅」ではなく「ビジネス上最適なカバレッジ」を論理的に抽出すること。
- [ ] **高度な技法の定着**: ペアワイズ法（All-pairs）、ドメイン分析（関連する変数の境界値）、状態遷移のNスイッチテストの実践的演習。
- [ ] **プロセス改善の視点**: 学んだテスト技法やドキュメント要件を、「どうすればLLMに理解・自動生成させられるか？」というAI連携のアーキテクチャ（プロンプト）に翻訳する。

## 6. Phase 1 学習計画 (TA 基礎構築フェーズ)
実務でテストマネージャー的役割を担いながら、AIを壁打ち相手として以下のドリルを推進する。
- シラバスを通読しつつ、基礎となる「第3章（テスト技法）」の高難易度シナリオを解く。
- 次に「第1章・第2章（プロセスとマネジメント）」をマネージャーの実務課題と紐づけて議論（K4適用）する。
- ドリルを通じて明らかになった「現場で使えるAIプロンプト化のヒント」を `02_Resources/TA_Techniques_Lab.md` 等に随時蓄積していく。
