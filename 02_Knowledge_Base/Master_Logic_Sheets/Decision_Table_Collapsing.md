# Master Logic Sheet: 決定テーブルの圧殺 (Collapsing)

## 1. 概念定義
決定テーブルにおいて、ある特定の条件が何であれ（TrueでもFalseでも）、他の条件が同じであれば同じアクションが発生する場合、そのルールを1つにまとめることができる。

- **用語**: ルールの圧殺（Collapsing）、あるいは統合。
- **表記**: 変化しても影響しない条件を「-」（ハイフン/Don't care）で表す。

## 2. 例
| 条件 | Rule 1 | Rule 2 | Rule 3 (統合後) |
| :--- | :--- | :--- | :--- |
| ログイン済 | Y | Y | Y |
| 有料会員 | Y | N | **-** |
| **アクション: 閲覧** | **実行** | **実行** | **実行** |

## 3. 注意点
- 圧殺を行うことで、理論上の最大ルール数（2^n）から見た「見かけ上のルール数」が減る。
- テキストベースのテストケースを減らす効果があるが、論理的な網羅性は失われない。
