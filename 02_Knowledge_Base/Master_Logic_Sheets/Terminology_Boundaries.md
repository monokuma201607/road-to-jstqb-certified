# Master Logic Sheet: 厳密な用語定義とプロセスの境界線
*(Created from Weakness Pattern: 2026-03-15 High Difficulty Mock Exam)*

現在の実力（偏差値65以上、正答率85%強）において、失点の原因は「考え方」ではなく「JSTQBが定める厳密な定義・境界線」のわずかなブレのみです。
以下の表を用いて、似た概念を明確に切り離してください。

## 1. 原則とプロセスの境界線

| 概念 A | 概念 B | 決定的な違い（境界線） |
| :--- | :--- | :--- |
| **欠陥の偏在** (パレートの法則) | **殺虫剤のパラドックス** (テストの劣化) | 「偏在」は**場所（集中する箇所）**の話。「殺虫剤」は**時間（同じテストを繰り返すと見つからなくなる）**の話。両者を混同させる選択肢に注意。 |
| **テスト「実装」**(Implementation) | **テスト「実行準備」**(Entry Criteria) | スケジュールに基づく**「割り当て」**は実装活動。実行に移れるかどうかは**「開始基準（Entry Criteria）」**を満たしたかどうかで判断される。 |
| **ファシリテーター** (モデレーター) | **マネージャー** | レビュー会議の「進行・仲裁」を行うのがファシリテーター。レビュー自体の「成功を保証」し、時間を割り当てるのはマネージャー。 |
| **プロダクトオーナー** (PO) | **テストアナリスト** | アジャイル環境で「ビジネス的な優先順位」を最終決定するのはPO。テストアナリストは「受け入れ基準の定義を支援」する。 |

## 2. 第4章：技法における定義の罠

| 概念 A | 概念 B | 決定的な違い（境界線） |
| :--- | :--- | :--- |
| 決定テーブルの**「圧縮」**(Collapsing) | なし | 圧縮の絶対条件は、「結果が同じ」であり、かつ**「たった1つの条件だけ」**が異なっていること。「少なくとも1つ異なる」という罠に注意する。 |
| **状態遷移** | **決定テーブル** | 状態遷移は**「過去の履歴（現在の状態）とイベント」**による振る舞いの変化。決定テーブルは**「現在の入力条件の論理的な組み合わせ」**による結果の決定。 |
