# JSTQB 克服マスターロジック (2026-03-03)

本日の失点パターンから導き出した、試験直前に見直すべき「黄金律」です。

## 1. 【技法】決定テーブルの「真の」最小列数
- **迷ったら**: 条件が2つなら $2^2 = 4$ 列、3つなら $2^3 = 8$ 列をまず書く。
- **集約の条件**: 「条件Aが No なら、条件Bが Yes でも No でもアクションが**全く同じ**」ときのみ、1列にまとめられる。
- **要注意**: アクションが同じでも、条件の意味として別物ならまとめないのが安全。

## 2. 【技法】判定網羅（決定網羅）のカウント法
- **ロジック**: True の矢印と False の矢印、すべてを1回以上通る。
- **並列/直列の罠**:
    - IF文が2つ並んでいる（直列）なら、1つのケースで `A:T, B:T`、もう1つで `A:F, B:F` を選べば、**2ケース**で網羅可能。
    - 「判定の数だけケースが必要」という思い込みを捨てる。

## 3. 【技法】状態遷移：最小パスの考え方
- **鉄則**: 「一回の実行（開始→終了）」で、どれだけ多くの遷移（矢印）を消化できるか。
- **ループの扱い**: 「A→B→A」という遷移があるなら、1つのケースで `A→B→A→C`（Cが終了）と動けば、3つの遷移を一気に網羅できる。

## 4. 【知識】テスト設計 vs テスト実装
- **設計 (Design)**: 要件を「テストケース（論理的、高水準）」に落とし込む。期待結果の特定はここ。
- **実装 (Implementation)**: テストケースを「実行可能なスクリプト」にする。データのロード、テストスイートの作成、実行順序の決定。
